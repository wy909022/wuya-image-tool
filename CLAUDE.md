# 无涯记账 APP — 项目文档

## 项目概述

- **项目名称**：无涯记账
- **产品定位**：个人日常支出记账工具
- **目标用户**：普通个人用户，用于记录和管理日常花销
- **运行平台**：Windows 桌面应用
- **项目路径**：`d:/WuyaApp`
- **开发时间**：2026年7月

---

## 技术栈（已确认）

| 技术组件 | 用途 |
|---------|------|
| Electron 27 | 桌面应用框架，将网页打包成 Windows .exe |
| Vue 3 (Composition API) | 前端 UI 框架 |
| electron-vite | 开发和构建工具 |
| sql.js | 本地 SQLite 数据库（纯 JS/WASM 实现，无需编译） |
| Element Plus | UI 组件库（按钮、表单、弹窗等） |
| ECharts | 图表库（统计页饼图） |
| dayjs | 日期处理库 |
| vue-router | 页面路由 |

## 核心功能需求

### ✅ 已完成
1. **记账功能**：添加支出记录 — 金额（元）、日期、一级/二级分类、备注（可选）
2. **账单列表**：按时间倒序展示所有记录，支持按月/分类筛选和删除
3. **统计面板**：按月份和分类查看支出汇总，包含饼图可视化
4. **数据持久化**：所有数据存储在本地 SQLite 数据库（`%APPDATA%/com.wuya.accounting/wuya-accounting.db`）

## ⚠️ 环境问题

### ELECTRON_RUN_AS_NODE 问题
当前系统设置了 `ELECTRON_RUN_AS_NODE=1` 环境变量，导致 Electron 以纯 Node.js 模式运行（API 不可用）。开发时使用 `npm run dev` 会自动清除该变量。

如果手动启动 Electron，需先执行：`unset ELECTRON_RUN_AS_NODE`

## ⭐ 开发约定（重要！必须遵守）

### 1. 技术决策规则（最高优先级）
> **用户在技术方面是小白，无法提供具体技术需求。因此，在整个项目开发过程中，遇到任何需要做技术决策的情况，Claude 必须先列出 2-4 个可行方案，用通俗易懂的语言（避免专业术语，或用类比解释）向用户说明每个方案的优劣势，然后让用户做出最终选择。Claude 可以给出推荐方案，但绝不能代替用户做决定。**
>
> 这条规则适用于但不限于以下场景：
> - UI 布局和交互方式的选择
> - 第三方库的选择
> - 功能实现方式的选择
> - 配色方案和视觉风格的选择
> - 任何有多个可行方案的技术点

### 2. 代码注释
- 所有代码注释使用**中文**
- 关键逻辑必须写注释说明

### 3. UI 组件
- 优先使用 Element Plus 组件库，保持界面统一

### 4. 数据存储
- 使用 sql.js（SQLite WASM 实现），数据库文件位于 `%APPDATA%/com.wuya.accounting/`
- 不收集、不上传任何用户数据

### 5. 文件组织
- 前端页面：`src/renderer/src/views/`
- Electron 主进程：`src/main/index.js`
- 预加载脚本：`src/preload/index.js`
- 通用组件：`src/renderer/src/components/`
